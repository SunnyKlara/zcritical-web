## 1. 北极星指标

> 一个团队不能有 3 个目标，只能有 1 个。

### Critical 的北极星

**让室内骑行训练的"沉浸感"成为可购买的产品**。

## 4. 决策原则（每次取舍时回到这里）

### 原则 1：质量 > 速度

- 不接受"先放着以后再说"
- 每个 PR 必须有：测试、文档、迁移指南
- 不允许 `// TODO` 进 main 分支

### 原则 2：明确 > 隐式

- 配置写在 yaml/env，不写在代码里
- 错误必须有 code + message + actionable hint
- 任何"魔术数字"必须有命名常量

### 原则 3：可观测 > 可调试

- 任何关键路径必须有 metrics + logs + traces
- 任何异常必须能在 Sentry 5 分钟内定位
- 任何用户报错必须能复现到具体 request_id

### 原则 4：安全是第一类要求

- 不是"上线后加"，是"每次写代码都想"
- 任何输入都是不可信的
- 任何输出都要 escape

### 原则 5：依赖最小化

- 引入新 npm 包要走 RFC 流程
- 优先用平台原生能力（fetch > axios，Web Crypto > bcryptjs）
- 关注 supply-chain 风险

## 5. 不做什么（Anti-Goals）

明确**不做**的事情，比明确做的事情同样重要：

- ❌ 不做"快速 MVP" — 不会牺牲质量换上线
- ❌ 不做大规模 SEO 投放 — 等内容打磨完再做
- ❌ 不做付费广告（直到 Phase 2）
- ❌ 不做 App Store / Google Play 上架（直到 v1.1）
- ❌ 不做多币种 / 多市场 — 先把 USD + 中英文做透
- ❌ 不做用户系统 — 当前购物只用邮箱
- ❌ 不做积分 / 会员体系 — 用产品质量留人，不靠套路

## 6. 成功标准（每个 Phase 必须达成才能进下一个）

### Phase 0 → Phase 1 准入门禁

全部 ✅ 才能进 Phase 1：

- [ ] 8 张架构图全部完成
- [ ] Lighthouse Performance ≥ 90 (mobile)
- [ ] OWASP ZAP 扫描 0 high / 0 critical
- [ ] k6 压测 100 RPS 下错误率 < 0.1%
- [ ] 所有路由 SSR/SSG 正确（无 hydration 错误）
- [ ] 41+ 测试 100% 通过 + 覆盖率门槛
- [ ] Staging 环境完整可用
- [ ] 自动备份脚本验证可恢复
- [ ] 灾备 runbook 演练成功
- [ ] FAQ ≥ 30 条 + Press kit 完成
- [ ] 1 个客户证言 / case study

### Phase 1 → Phase 2 准入门禁

- [ ] 上线 7 天内无 P0/P1 事故
- [ ] 真实用户首次完成购买
- [ ] PostHog 数据流通畅
- [ ] Sentry 0 unhandled critical errors

## 10. 工作节奏

### 每日（你不需要每天来，但我每天都在跑）

- 我自动推进具体任务，每天产出 1-3 个 commit
- 你回来时看 `docs/DAILY-LOG.md` 知道今天做了什么
- 卡住的任务（需要你决定 / 提供凭证）记录在 `docs/COLLAB.md`

### 每周

- 我写一份 weekly report 进 `docs/weeks/W{N}-REPORT.md`
- 你 review，给方向反馈

### 每月

- 月末做一次 retrospective 进 `docs/monthly/M{N}-RETRO.md`
- 调整下个月计划

### 每个 Phase 结束

- 准入门禁全部检查
- 写 Phase 总结
- 决定是否进入下个 Phase
